## 第一梯队：投入小、体验提升明显

### 1. Mock 模式回复扩展（P1）

- 当前没有 API Key 时回复很单调，`persona.py` 只有 4 条固定模板
- 改动量小（扩展模板表 + 根据 intent/emotion 随机组合），但 demo 体验提升很大
- 适合展示给他人看，是"没有 API Key 也能玩"的关键
- 预估工时：半天

### 2. 服务端记忆 API（P3）

- 当前 localStorage 刷新换浏览器就丢
- SQLite 存储，改动量中等，但让宠物真正"记住你"
- 是长期陪伴感的基础
- 需要实现：`/api/memory` GET/POST/DELETE，SQLite 表结构
- 预估工时：1-2 天

### 3. 记忆编辑 UI（P3）

- 配合服务端记忆，让用户看到/删除/修改宠物记住的东西
- 透明感 = 信任感，产品体验质变
- 前端新增 `MemoryPanel` 增强版，支持编辑和删除
- 预估工时：1 天

---

## 第二梯队：有深度、值得投入

### 4. LLM Provider 适配层（P4）

- 当前 `root_brain.py` 直接用 OpenAI 兼容接口
- 加一层 provider 抽象，支持 DeepSeek / Gemini / Kimi / GLM 无缝切换
- 架构价值高，后续换模型零成本
- 实现方式：`services/llm_provider.py`，根据 `LLM_PROVIDER` 环境变量选择适配器
- 预估工时：半天

### 5. Agent Trace 增强（P4）

- 当前 trace 已有 CoT 推理步骤 + 验层信息
- 可以做"普通模式"和"开发者模式"两种展示
- 让用户看到宠物"在想什么"，产品差异化
- 前端 `AgentTracePanel` 增加折叠/展开和详细程度切换
- 预估工时：半天

### 6. Stop phrase 优化（P1）

- 当前用 broad substring matching，容易误触发
- 改成更精准的会话结束检测（意图分类 + 上下文判断）
- 影响免提会话的稳定性
- 预估工时：半天

---

## 第三梯队：长期价值、但可以缓一缓

| 功能 | 为什么可以缓 |
|---|---|
| PWA / Electron | 先把核心体验打磨好再包装 |
| Google ADK 集成 | 当前单 Agent 够用，过早引入增加复杂度 |
| 向量记忆检索 | 记忆量不够大时，简单列表注入足够 |
| 工具调用层 | 先让对话体验稳定，再加外部能力 |
| 部署 | 本地能跑就行，展示时再考虑 |
| 语音语言控制 | 当前中文够用，英文支持优先级低 |
| 低置信度启发式调优 | 需要真实使用数据积累，现在调为时过早 |
| 记忆合并去重 | 记忆量少时问题不大，等积累多了再做 |
| 记忆导入导出 | 等服务端记忆稳定后再加 |
| State API 方向 | 当前前端持有状态能跑，不急着改 |

---

## 建议开发顺序

```text
1. Mock 模式扩展       → 半天搞定，立刻提升 demo 体验
2. 服务端记忆（SQLite） → 1-2 天，宠物真正有记忆
3. LLM Provider 适配层 → 半天，架构收益高
4. 记忆编辑 UI         → 1 天，产品体验质变
5. Agent Trace 增强    → 半天，差异化展示
6. Stop phrase 优化    → 半天，稳定性提升
```

---

## 不建议现在做的

- **复杂 3D / Live2D**：偏离终端风格定位
- **用户登录/支付/社交**：MVP 阶段不需要
- **摄像头识别/自动控制电脑**：偏离核心体验
- **大规模工具调用**：先让对话体验稳定
