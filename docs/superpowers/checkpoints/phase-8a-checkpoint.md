## Architecture

### Module Responsibilities

| Module | File | Responsibility |
|--------|------|----------------|
| Intent Agent | `agents/intent.py` | Classifies user message: greeting, sad, question, default |
| Emotion Agent | `agents/emotion.py` | Maps intent to allowed emotion |
| Action Agent | `agents/action.py` | Maps intent to allowed action |
| State Agent | `agents/state_delta.py` | Computes state delta per action rules |
| Memory Agent | `agents/memory_decision.py` | Decides whether to save message as memory |
| Persona Agent | `agents/persona.py` | Generates reply text (mock mode) |
| Root Brain | `agents/root_brain.py` | Orchestrates modules, handles LLM calls, validation |

### Two Execution Paths

**Mock path** (no `LLM_API_KEY`):
```
message → intent → emotion → action → state_delta → memory → persona → ChatResponse
```
All six modules run sequentially with rule-based logic. Each module produces its own trace entry.

**LLM path** (with `LLM_API_KEY`):
```
message → DeepSeek LLM → JSON → validation → trace inference → ChatResponse
```
Single LLM call, then trace entries are generated by inspecting the LLM output using the intent module for classification.

### Trace Output (both paths)

```json
"trace": [
  {"module": "intent_agent", "message": "detected greeting"},
  {"module": "emotion_agent", "message": "user mood positive"},
  {"module": "action_agent", "message": "selected wake"},
  {"module": "memory_agent", "message": "no stable memory"},
  {"module": "root_agent", "message": "response assembled via LLM"}
]
```

## Module Details

### intent.py
- Regex-based keyword matching (Chinese + English)
- Categories: greeting, sad, question, default
- Zero dependencies — pure function

### emotion.py
- Maps intent to one of 7 allowed emotions
- Takes intent string + pet state context
- Returns emotion string

### action.py
- Maps intent to one of 10 allowed actions
- Takes intent + emotion + pet state
- Returns action string

### state_delta.py
- Per-action delta rules (10 action types)
- Returns `StateDelta` Pydantic model
- Values pre-clamped to [-10, +10]

### memory_decision.py
- Regex patterns for storable personal info (name, preferences, goals, habits)
- Returns `Memory` with `should_save` bool and content
- Content truncated to 200 chars

### persona.py
- Template replies keyed by intent
- Used in mock mode only
- LLM mode uses the LLM's own reply

### root_brain.py
- Imports all modules
- `_mock_response()`: runs all 6 modules sequentially
- `_call_llm()`: single LLM call + validation + trace inference
- `_infer_llm_trace()`: generates module-style trace from LLM output
- `generate_response()`: public entry point, routes mock vs LLM
- `fallback_response()`: glitch response on failure
- `_validate_llm_response()`: strict validation with action/emotion whitelists
