## Architecture: RootBrain-Orchestrated Multi-Agent

NEON PAW uses a self-developed orchestrator (`root_brain.py`) that coordinates seven agent modules. This is NOT a Google ADK runtime — it is custom Python orchestration designed to be ADK-compatible if migration is desired later.

### Module Map

```
root_brain.py (orchestrator)
  ├── intent.py           → classify user message (greeting/sad/question/default)
  ├── emotion.py          → map intent to pet emotion
  ├── action.py           → map intent to pet action
  ├── state_delta.py      → compute per-action state changes
  ├── memory_decision.py  → decide whether to save memory
  ├── persona.py          → generate mock-mode reply text
  └── (LLM call)          → DeepSeek API via AsyncOpenAI
```

### Two Execution Paths

**Mock path** (no `LLM_API_KEY`):
```
message → intent → emotion → action → state_delta → memory → persona → ChatResponse
```
All six modules run sequentially with rule-based logic.

**LLM path** (with `LLM_API_KEY`):
```
message → DeepSeek LLM → JSON → validation → trace inference → ChatResponse
```
Single LLM call, then trace entries generated by inspecting the response.

Both paths produce identical `ChatResponse` structure with 5-module trace entries.

---

## Key Design Decisions

1. **RootBrain is the coordinator.** Not Google ADK, not LangChain, not any external framework.
2. **DeepSeek is the LLM provider.** Uses OpenAI-compatible API via `AsyncOpenAI`.
3. **Modules are pure Python functions.** No class inheritance, no agent framework, no decorators.
4. **ADK migration is optional.** The module boundaries map cleanly to ADK sub-agents, but this is not required.
5. **Mock mode is fully functional.** All six modules produce meaningful output without any API key.

## ADK Readiness (Future Option)

The current architecture can migrate to Google ADK if needed:

| Current Module | ADK Equivalent |
|---|---|
| `intent.py` → `detect_intent()` | ADK sub-agent or function tool |
| `emotion.py` → `detect_emotion()` | ADK sub-agent or function tool |
| `action.py` → `select_action()` | ADK sub-agent or function tool |
| `state_delta.py` → `compute_state_delta()` | ADK function tool |
| `memory_decision.py` → `decide_memory()` | ADK function tool |
| `persona.py` → `generate_reply()` | ADK sub-agent |
| `root_brain.py` → orchestrator | ADK `Runner` + `LlmAgent` |

Migration path:
1. Install `google-adk` and `litellm` (for DeepSeek support)
2. Create `agents/adk_agent.py` with `LlmAgent` + `Runner`
3. Add `USE_ADK` env flag to `config.py`
4. Add ADK tier to `generate_response()` before DeepSeek tier
5. Existing mock + DeepSeek pipelines remain as fallback

This is documented in the cancelled Phase 8B spec for reference.

## How to Run

### Backend (mock mode — no API key)

```bash
cd backend
source venv/bin/activate
LLM_API_KEY="" uvicorn app.main:app --reload --port 8000
```

### Backend (DeepSeek mode)

```bash
cd backend
source venv/bin/activate
# Ensure .env has LLM_API_KEY set
uvicorn app.main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
# Opens at http://localhost:3000
```
